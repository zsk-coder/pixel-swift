# 设计图还原 Workflow

当用户提供设计图（截图/标注）需要还原为 Vue 组件时，按以下步骤执行：

## 1. 分析设计图

仔细分析用户提供的设计图，提取以下信息并**列出**确认：

- **布局结构**：flex/grid、排列方向、嵌套层级
- **颜色**：背景色、文字色、边框色、渐变（优先匹配 `tailwind.config.ts` 中已有 token）
- **字体**：字号、字重、行高
- **间距**：元素间 margin/padding、容器内边距
- **圆角/阴影**：border-radius、box-shadow
- **交互状态**：hover、active、disabled、dark mode
- **响应式**：移动端和桌面端的差异

## 2. 确认技术方案

向用户确认：

- 这是新页面还是现有组件的修改？
- 对应哪个路由/组件文件？
- 有没有已有的相似组件可以复用？
- 设计图中的文案是写死还是用 i18n？

## 3. 编写代码

按以下原则编码：

### 样式优先级

1. **优先使用项目已有的 Tailwind token**（如 `text-brand`、`bg-surface-card`、`rounded-card`）
2. **其次使用项目 SCSS 工具类**（如 `.btn-primary`、`.card`、`.content-wrapper`）
3. **最后才写自定义样式**（scoped SCSS/Tailwind 任意类）

### 代码规范

- 使用 `<script setup lang="ts">` + Composition API
- 文案使用 `useI18n()` 的 `t()` 函数，同步更新 locale 文件
- 组件拆分粒度：一个设计图区块 = 一个组件（超过 150 行考虑拆分）
- 图片素材用 `generate_image` 工具生成（不用占位符）
- 所有交互元素添加无障碍属性（aria-label、role）

### 响应式

- 移动端优先（默认样式 = 手机端）
- 用 `sm:`、`md:`、`lg:` 处理桌面适配
- 关键断点：`sm: 640px`、`md: 768px`、`lg: 1024px`

## 4. 视觉校验

代码写完后：

1. 在浏览器中打开对应页面
2. 截图与设计图对比，检查：
   - 间距是否一致
   - 颜色是否匹配
   - 字体大小是否正确
   - 暗色模式是否正常
   - 移动端布局是否正确
3. 如有偏差，调整后再次截图确认

## 5. 输出确认

向用户展示：

- 还原后的页面截图
- 与设计图的差异说明（如有）
- 修改的文件列表
