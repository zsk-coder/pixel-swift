# 📐 PixelSwift Smart Check — 设计规格文档

> **给设计师**：这是一个 AI 驱动的电商产品图合规检查工具。用户拖入产品图 → AI 自动分析是否符合 Amazon/Shopify 等平台要求 → 一键修复不合规的图片。

---

## 1. 产品背景

### 我们在做什么？

跨境电商卖家上架产品时，经常因为图片不合规被平台退回（尺寸不够、文件太大、格式不对）。他们需要一个工具：**拖入图片 → AI 帮我检查 → 帮我修好 → 直接下载上架**。

### 核心理念

- **不是聊天机器人**，是一个**审计报告工具**
- AI 分析结果以**流式报告**呈现（文字逐行出现，类似 Cursor 写代码的效果）
- 图片**不离开用户浏览器**，隐私安全是核心卖点

### 竞品参考（设计方向）

| 产品                  | 可参考的点                               |
| --------------------- | ---------------------------------------- |
| **Google Lighthouse** | 评分报告的视觉结构（通过/警告/失败分色） |
| **Grammarly**         | "发现问题 → 一键修复"的交互模式          |
| **Vercel Build Log**  | 流式输出日志的 UI 表现                   |
| **GitHub Actions**    | 步骤展开/折叠、状态图标的设计            |

## 5. 主工具页 — 5 个状态

### 状态 1：空状态（初始进入）

**用户看到的：**

- 页面标题 + 副标题
- 平台选择器（下拉菜单）
- 大面积拖拽区域
- 隐私承诺

**UI 规格：**

| 要素         | 内容                                                                                   |
| ------------ | -------------------------------------------------------------------------------------- |
| 标题         | "AI Product Image Compliance Checker"                                                  |
| 副标题       | "Check if your product images meet Amazon, Shopify, Etsy requirements — powered by AI" |
| 平台选择器   | 下拉菜单：Amazon 主图 / Amazon 副图 / Shopify / Etsy / eBay                            |
| 拖拽区       | 居中，虚线边框，图标 + 文案                                                            |
| 拖拽区文案   | "拖拽产品图到这里，或点击上传"                                                         |
| 拖拽区副文案 | "支持 JPEG, PNG, WebP, GIF · 最多 20 张 · 单张 ≤ 50MB"                                 |
| 隐私标识     | 🔒 "AI 只分析图片的数字参数，图片不会上传到任何服务器"                                 |
| 底部         | 已支持平台 Logo（Amazon, Shopify, Etsy, eBay）                                         |

**交互：**

- 拖入时：边框变品牌色实线 + 背景淡色 + "释放以开始"
- 未选平台就拖入：提示"请先选择目标平台"

### 状态 3：AI 分析中（⭐ 最重要的页面）

**报告面板设计要点：**

> ⚠️ **不是聊天气泡！** 是 Terminal / 日志面板 / GitHub Actions 步骤的结构化输出。

| 要素     | 规格                              |
| -------- | --------------------------------- |
| 面板风格 | 深色背景 or 浅灰背景              |
| 流式动效 | 文字逐行淡入出现                  |
| 进度     | 顶部 "分析中... 5/20 张" + 进度条 |

**每张图的报告结构：**

```
[状态图标] [文件名] — [总体判定]
   ├─ [检查项]: [通过/不通过] + [信息]
   ├─ [检查项]: [不通过] + [AI 建议]
   │  ⚡ 已安排修复: [动作说明]
   └─ [检查项]: ...
```

**示例输出：**

```
📋 正在检索 Amazon 2026 主图规范...

✅ product-01.jpg (1200×1200, 560KB)
   尺寸 ✅ | 大小 ✅ | 格式 ✅ | 背景 ✅

❌ product-02.png (800×600, 3.2MB) — 3 个问题
   ├─ 尺寸：800×600 → 低于要求 1000×1000
   │  ⚡ 将放大至 1000×1000
   ├─ 大小：3.2MB → 建议 < 1MB
   │  ⚡ 将压缩至 900KB
   └─ PNG 无透明通道 → 转 JPEG 可减少 60%
      ⚡ 将转换为 JPEG

⚠️ product-03.jpg (1000×1000, 890KB) — 1 个警告
   └─ 背景偏灰 (#F0F0F0)：Amazon 要求纯白
      💡 建议使用抠图工具处理

▶ 分析中... 3/8 张
```

### 状态 4：分析完成 + 修复计划

| 要素     | 内容                                                     |
| -------- | -------------------------------------------------------- |
| 摘要     | "分析完成：20 张 · ✅ 通过 12 · ⚠️ 警告 3 · ❌ 不合规 5" |
| 修复列表 | 勾选框列表（默认全选），用户可取消单项                   |
| 主按钮   | "🔧 一键执行全部修复 (8 项)"                             |
| 次要按钮 | "跳过修复，仅下载原图"                                   |

**修复列表示例：**

```
☑ resize product-02.png → 1000×1000
☑ compress product-02.png → 900KB
☑ convert product-02.png → JPEG
☑ compress product-05.jpg → 800KB
☑ strip_exif product-07.jpg（清除GPS）
☐ resize product-11.png → 1000×1000   ← 可取消
```

---

### 状态 5：修复完成 + 下载

**修复中：**

- 进度条 + "修复中 3/8 项..."
- 当前操作："正在压缩 product-02.png..."
- 完成项打勾变绿 + 大小变化 "3.2MB → 890KB ↓73%"

**修复完成：**

- "✅ 8 项修复已完成！"
- 主按钮："📥 下载修复后的图片 (ZIP)"
- 统计："总共节省 4.8MB · 5 张图片已达标"
- 次要按钮："继续检查更多图片"
- 分享引导："觉得有用？分享给同事"

## 9. 边界场景

| 场景         | 处理                         |
| ------------ | ---------------------------- |
| 0 张图       | 按钮灰色不可点               |
| > 20 张      | 提示分批                     |
| 单图 > 50MB  | 红色边框 + "文件过大"        |
| 非图片文件   | 忽略 + 提示格式要求          |
| 额度用完     | 按钮变灰 + "明天 0:00 重置"  |
| AI 出错/超时 | "分析遇到问题" + 重试按钮    |
| 断网         | "需要网络连接进行 AI 分析"   |
| 全部合规     | 🎉 庆祝动效 + "可直接上架！" |
