# 📄 PixelSwift Smart Check — 产品需求文档 (PRD)

> **版本**: v4.0 AI-First  
> **最后更新**: 2026-04-17  
> **核心决策**: LangChain Agent 统一驱动分析决策，浏览器端仅做特征提取 + 本地修复

---

## 1. 产品愿景

### 一句话定位

> **AI 驱动的电商产品图合规审计员** — 拖入产品图 + 选择平台，AI Agent 自动诊断问题、给出修复方案并一键执行。

### 类比定位

| 概念                        | 说明                                       |
| --------------------------- | ------------------------------------------ |
| ❌ 不是 "图片版 Lighthouse" | Lighthouse 受众是开发者，电商卖家不看评分  |
| ✅ 是 "图片版 Grammarly"    | 用户传入内容 → 自动发现问题 → 一键接受修复 |
| ❌ 不是聊天机器人           | 没有对话输入框，是结构化报告 + 一键操作    |

### 为什么现在做？

| 时机因素                         | 说明                                                   |
| -------------------------------- | ------------------------------------------------------ |
| **PixelSwift 已有 75% 基础设施** | 压缩/转换/resize/Worker/WASM 全部可复用                |
| **SEO 长尾词空缺巨大**           | "amazon image requirements" 类词竞争度中低             |
| **跨境电商持续增长**             | 全球跨境电商 2026 预计 $7.9T                           |
| **AI 技术栈对简历的加分**        | LangChain/RAG/Function Calling 是 2025-2026 求职硬通货 |

## 3. 用户旅程

```
① 到达 /smart-check 页面
        ↓
② 选择目标平台（Amazon/Shopify/Etsy/eBay）
        ↓
③ 拖入产品图（最多 20 张，无需登录即可加载）
        ↓
④ 浏览器瞬间提取图片特征（< 100ms，用户无感知）
        ↓
⑤ 点击"启动 AI 分析" → 弹出 Google 登录（首次）
        ↓
⑥ AI Agent 流式分析（2-5 秒）
   → 逐张输出诊断报告（类似 Terminal 日志）
   → 自动安排可修复项
        ↓
⑦ 一键执行修复（浏览器本地，3-10 秒）
        ↓
⑧ ZIP 自动下载修复后的图片
```

### 关键体验指标

| 指标               | 目标       |
| ------------------ | ---------- |
| 图片加载到能点分析 | < 3 秒     |
| 特征提取           | < 100ms/张 |
| AI 首字输出 (TTFB) | < 1000ms   |
| 20 张图完整分析    | < 15 秒    |
| 一键修复 20 张     | < 30 秒    |

## 4. 功能范围

### ✅ MVP 做什么

| 功能             | 说明                                                      |
| ---------------- | --------------------------------------------------------- |
| **平台选择**     | 支持 Amazon 主图/副图、Shopify、Etsy、eBay                |
| **图片特征提取** | 浏览器端提取尺寸/大小/格式/EXIF/模糊度/背景白度           |
| **AI 合规诊断**  | LangChain Agent + RAG 对比平台规范，流式输出报告          |
| **自动修复**     | AI 通过 Function Calling 安排 压缩/缩放/格式转换/EXIF清除 |
| **一键批量修复** | 浏览器本地执行修复，ZIP 打包下载                          |
| **Google 登录**  | Supabase Auth，仅 Google OAuth                            |
| **每日配额**     | 免费用户 10 次/天                                         |
| **SEO 着陆页**   | 4 个平台着陆页 × 8 语言 = 32 页                           |

### ❌ MVP 不做什么

| 砍掉的         | 原因                              |
| -------------- | --------------------------------- |
| 聊天对话框     | 用户不需要"聊天"，需要"报告+修复" |
| AI 抠图/换白底 | 计算成本过高，MVP 仅文字提示      |
| 网站 URL 审计  | 需后端代理，架构复杂              |
| 邮箱登录       | 跨境卖家 Google 渗透率 ~100%      |
| 付费订阅       | 先验证 PMF，MAU > 5K 再上         |
| PDF 报告导出   | 低频需求，后续迭代                |

## 5. 平台合规规则集

| 平台            | 图片类型  | 最小尺寸        | 最大文件 | 允许格式             | 白底要求 |
| --------------- | --------- | --------------- | -------- | -------------------- | -------- |
| **Amazon 主图** | 产品主图  | 1000×1000       | 10MB     | JPEG, PNG, GIF, TIFF | 纯白     |
| **Amazon 副图** | 细节/场景 | 1000×1000       | 10MB     | JPEG, PNG, GIF, TIFF | 不要求   |
| **Shopify**     | 产品图    | 2048×2048(推荐) | 20MB     | JPEG, PNG, GIF       | 不要求   |
| **Etsy**        | 商品图    | 2000×2000(推荐) | 10MB     | JPEG, PNG, GIF       | 不要求   |
| **eBay**        | 产品图    | 500×500         | 12MB     | JPEG, PNG            | 不要求   |

### AI 检查项

| 检查项   | 说明                    | 可自动修复    |
| -------- | ----------------------- | ------------- |
| 文件大小 | 是否超过平台上限        | ✅ 压缩       |
| 尺寸     | 宽高是否满足最低要求    | ✅ 放大/缩小  |
| 格式     | 是否被平台支持          | ✅ 格式转换   |
| 格式优化 | PNG 无透明 → 建议 JPEG  | ✅ 格式转换   |
| DPI      | 是否适合用途            | ✅ 修改元数据 |
| EXIF/GPS | 是否包含隐私数据        | ✅ 清除       |
| 模糊度   | 图片是否模糊            | ❌ 仅提示     |
| 背景白度 | 是否纯白（Amazon 主图） | ❌ 仅提示     |

## 8. 核心隐私卖点

> **AI 诊断不传原图，100% Client-side 隐私保护**

- 浏览器端提取图片数字特征（尺寸/大小/格式/模糊度等）
- 仅向 AI 发送特征 JSON，不发送原图
- 所有图片修复操作在浏览器本地执行（WASM）
- 页面关闭后图片从内存中彻底清除

## 10. 风险与应对

| 风险              | 应对                                                 |
| ----------------- | ---------------------------------------------------- |
| AI 分析结果不准确 | System Prompt 强约束 + RAG 结构化规则（不做PDF切片） |
| LLM API 成本超支  | 不传原图（省95% Token）+ gpt-4o-mini + 每日限额      |
| 模糊检测误判      | 保守阈值，提示"可能模糊"而非"确定模糊"               |
| SEO 起效慢        | 社群推广为主（Reddit/Facebook），SEO 为长期引擎      |
| 竞品抄袭          | 隐私差异化 + 多语言 SEO 先发优势                     |
