# Google Search Console & Analytics 使用指南

> 适用于 PixelSwift 项目，面向新手的核心功能指南。

---

## 一、它们分别是什么？

| | Google Search Console 🔍 | Google Analytics 📊 |
|---|---|---|
| **一句话** | 监控网站在**搜索引擎中的表现** | 监控**用户进入网站后的行为** |
| **比喻** | 你的店铺招牌在街上吸引了多少人 | 顾客进店后逛了哪里、待了多久 |
| **地址** | [search.google.com/search-console](https://search.google.com/search-console/) | [analytics.google.com](https://analytics.google.com/) |

---

## 二、Google Search Console — 重点关注

### 1. 效果报告 ⭐ 最重要

**位置**：左侧菜单 → 效果（搜索结果）

这是你最常看的页面，核心指标：

| 指标 | 含义 | 关注点 |
|------|------|--------|
| **总点击次数** | 用户从 Google 搜索点进你的网站的次数 | 越多越好，反映实际流量 |
| **总展示次数** | 你的网站出现在搜索结果中的次数 | 展示多但点击少 → 标题/描述需要优化 |
| **平均点击率 (CTR)** | 点击次数 ÷ 展示次数 | 低于 2% 需要优化标题和描述 |
| **平均排名** | 你的页面在搜索结果中的位置 | 目标：进前 10（第一页） |

**实用操作**：
- 点击 **"查询"** 标签 → 看用户通过哪些关键词找到你
- 点击 **"页面"** 标签 → 看哪些页面获得最多流量
- 筛选 **日期范围** → 对比不同时期的增长趋势

### 2. 网页索引

**位置**：左侧菜单 → 网页索引（索引编制）

| 状态 | 含义 | 需要操作吗？ |
|------|------|------------|
| ✅ 已编入索引 | Google 已收录，会出现在搜索结果中 | 不需要，正常 |
| ❌ 未编入索引 | Google 知道但没收录 | 检查原因，可能需要优化 |
| 🚫 被 robots.txt 排除 | 你主动告诉 Google 不要收录 | 确认是否有误排除 |

**建议**：每周看一次，确保重要页面都被收录。

### 3. 站点地图

**位置**：左侧菜单 → 站点地图

你已经提交了 `sitemap_index.xml`，这里可以看到：
- 站点地图是否提交成功
- Google 发现了多少个网址
- 上次读取时间

**状态说明**：
- ✅ 成功 → 正常
- ❌ 无法抓取 → 刚提交的话等 1-2 天再看
- ⚠️ 有错误 → 需要排查具体原因

### 4. 网址检查工具

**位置**：顶部搜索框

输入某个 URL（如 `https://pixelswift.site/converter`），可以查看：
- 该页面是否被 Google 收录
- 最后抓取时间
- 是否有问题

**技巧**：新发布页面后，可以在这里 **请求编入索引**，加快收录速度。

### 5. 核心网页指标

**位置**：左侧菜单 → 体验 → 核心网页指标

这是 Google 对你网站性能的评估：

| 指标 | 意思 | 达标标准 |
|------|------|---------|
| **LCP** (最大内容渲染) | 页面主内容加载速度 | ≤ 2.5 秒 |
| **INP** (交互响应性) | 用户点击后的响应速度 | ≤ 200 毫秒 |
| **CLS** (布局偏移) | 页面加载时内容是否跳动 | ≤ 0.1 |

**三个都是绿色 = 优秀**，Google 会给予排名加分。

## 三、Google Analytics — 重点关注

### 1. 实时报告

**位置**：左侧菜单 → 报告 → 实时

查看**当前时刻**有多少人在访问你的网站。适合在：
- 刚部署后验证 GA 是否正常工作
- 发布新内容/做推广时观察即时效果

### 2. 流量获取 ⭐ 最重要

**位置**：报告 → 流量获取 → 流量获取概览

查看用户**从哪里来**：

| 来源 | 含义 |
|------|------|
| **Organic Search** | 通过 Google 搜索进来（SEO 流量） |
| **Direct** | 直接输入网址访问 |
| **Referral** | 从其他网站的链接点进来 |
| **Social** | 从社交媒体进来 |

**关注点**：Organic Search 占比越高，说明 SEO 效果越好。

### 3. 互动度 — 页面和屏幕

**位置**：报告 → 互动度 → 页面和屏幕

查看**每个页面的表现**：

| 指标 | 含义 | 关注点 |
|------|------|--------|
| **浏览量** | 页面被看了多少次 | 哪些页面最受欢迎 |
| **用户数** | 有多少独立用户访问 | 区分刷新和真实用户 |
| **平均互动时长** | 用户在该页面停留多久 | 太短说明内容不吸引人 |

### 4. 用户属性

**位置**：报告 → 用户属性 → 概览

了解你的用户是谁：
- **国家/地区**：用户来自哪里
- **语言**：用什么语言访问（对你的多语言站很有价值）
- **设备**：手机 vs 电脑

### 5. 技术详情

**位置**：报告 → 技术 → 概览

了解用户使用什么设备和浏览器访问你的网站，帮助你决定兼容性优先级。

## 四、日常检查清单

### 每天（2 分钟）
- [ ] 打开 Analytics → **实时** → 确认有数据在收集

### 每周（10 分钟）
- [ ] Analytics → **流量获取** → 查看本周总流量趋势
- [ ] Analytics → **页面和屏幕** → 查看哪些页面最受欢迎
- [ ] Search Console → **效果报告** → 查看搜索关键词和点击趋势

### 每月（20 分钟）
- [ ] Search Console → **网页索引** → 确认重要页面都被收录
- [ ] Search Console → **核心网页指标** → 检查网站性能
- [ ] Analytics → **用户属性** → 了解用户来源国家和设备
- [ ] 对比本月 vs 上月数据，看增长趋势

## 五、常见问题

### Q: 数据要多久才能看到？
- **Analytics 实时报告**：部署后几分钟就能看到
- **Analytics 完整报告**：24-48 小时后
- **Search Console**：首次提交后通常 2-5 天，完整数据需要 1-2 周

### Q: 为什么 Search Console 显示的点击数和 Analytics 不一样？
这是正常的。Search Console 只统计 Google 搜索来的流量，Analytics 统计所有流量。

### Q: 本地开发 localhost 会不会被统计？
不会。`nuxt-gtag` 默认只在生产环境生效，localhost 的访问不会被统计。

### Q: 怎么知道 SEO 有没有效果？
最直观的指标：Search Console → 效果报告 → 观察 **总点击次数** 和 **总展示次数** 是否在增长。
